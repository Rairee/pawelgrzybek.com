Does JavaScript keep you too busy to care about CSS? Here are recent features you want to know about!

Published: 2019.11.27 | 7 minutes read

For the last few years, I have been busy exploring the world of JavaScript to the point that I neglected my passion for CSS. I am sure there are more people like me out there. I owe CSS a debt so I curated a list of recent features that you may want to know about if you didn’t follow this space for a little while.

  • CSS custom properties (CSS variables)
  • CSS Grid Layout
  • Variable fonts
  • CSS Scroll Snap
  • CSS Logical Properties and Values
  • Position sticky
  • Houdini
  • Feature query with @supports
  • Preprocessors and the CSS-in-JS drama
  • Extra stuff

CSS custom properties (CSS variables) #

We got used to assigning reusable values to variables using pre-processors like Sass, LESS and Stylus. This feature became popular to the point that CSS Custom Properties became a thing and the browsers support nowadays is satisfactory. I published “CSS Custom Properties explained” around three years ago where I go in-depth about this feature and show some practical example.

:root {
  --cool-color: hotpink;
}

a {
  color: var(--cool-color);
}

a:hover {
  --cool-color: lime;
}

CSS Grid Layout #

Layout is a thing that CSS has never been good at. From terrible hack using tables, neither nice use of floats through to more understandable flex layout. We finally got a legit way of doing it right — CSS Grid Layout. I published “Let’s get into the basics of CSS Grid Layout Model” in 2015 when this specification was slowly shaping out. Today we have a great browser adoption for CSS Grid and it gives me a wide smile on my face because I see more and more websites using it. It is close to impossible to show you a snippet that works as a holistic explainer because the specification is humongous. If you want to dip your toes into CSS Grid I cannot recommend enough “CSS Grid” course by Wes Bos. For people interested in the subject I recommend to follow Rachel Andrew and Jen Simmons on Twitter. If you are after lazy copy-and-paste reference, “A Complete Guide to Grid” by CSS Tricks is everything you need. Let’s have a quick look at a basic example.

.wrapper {
  display: grid;
  grid-gap: 1rem;
  grid-template-columns: 2fr 1fr;
}

.header {
  grid-area: 1 / 1 / 2 / 3;
}

.main {
  grid-area: 2 / 1 / 3 / 2;
}

.sidebar {
  grid-area: 2 / 2 / 3 / 3;
}

.footer {
  grid-area: 3 / 1 / 4 / 3;
}

Variable fonts #

Since the inception of the web platform, we have been able to use a web-safe subset of font families. This became a design restriction, so we got support for custom typefaces via @font-face at-rule. CSS Fonts Module Level 4 adds a concept of Variable Fonts that puts custom fonts to the whole new level. Rather than fetching separate font file for every width, weight, or style, we get many different variations of a typeface incorporated into a single file. Performance without compromising design requirements at the same time — yes, please! To control basic typeface characteristics (weight, width, slant, italic, and optical size) we can use one of the registered axes. On top of that typographers can register custom variation axis that gives them almost infinite possibilities. Jason Pamental maintains an amazing resource titled “Variable Fonts: What web authors need to know” which is the best guide to use Variable Fonts. Presented on CSSconf EU 2019 by the same author “Modern Typographic Systems With Variable Fonts” is very informative talk for sexy typography fans. Time for quick example and little interactive demo.

@font-face {
  font-family: Inter;
  src: url('Inter.woff2');
}

body {
  font-family: Inter;
  font-size: calc(var(--size) * 1rem);
  line-height: 1.5;
  font-variation-settings: 'wght' 300, 'slnt' -10;
}

CSS Scroll Snap #

Historically we have been throwing tons of JavaScript on our web projects to achieve some sort of scroll snapping. You can partially blame me for that because I am the creator of one of the most popular carousel libraries out there — Siema.js. Let me tell you a thing — do not use it! All the JavaScript solution come with some sort of performance and accessibility drawbacks. CSS Scroll Snap is a native implementation that solves a majority of carousel-obsessed designers requirements. Even if your projects requirement is outside of boundaries of browsers support range for CSS Scroll Snap you don’t need to worry too much. It gracefully falls back to regular scroll experience without breaking your layout whatsoever. Quick demo time.

.app {
  overflow-y: scroll;
  scroll-snap-type: x mandatory;
}

.app__section {
  scroll-snap-align: start;
}

CSS Logical Properties and Values #

CSS Logical Properties and Values enforce authors to control layout through logical, rather than physical direction and dimension mappings. It vastly improves working on projects that support multiple writing modes. For example, border-inline-start in horizontal-tb draws a border on the left-hand side of an element but for vertical-lr it will appear on the bottom of a node. This is a well-thought system designed with accessibility and internationalization in mind. I cannot recommend enough “CSS Logical Properties” by Adrian Roselli – easily the best resource out there. This CodePen taken from Adrian’s post is a fantastic visual explainer.

Position sticky #

I can’t count how many approaches of building a sticky header element I came across during my career. They always relied on window sizes combined with scroll event. Later on, IntersectionObserver improved a bit — if you are not familiar with this fantastic interface, you should give my “The Intersection Observer API explained” a read. Wouldn’t it be cool to achieve this effect using just CSS? Good news is coming — it is possible now and the browser support for position: sticky is sufficient for some. Look how simple it is!

section {
  position: sticky;
  top: 1rem;
}

Houdini #

This one deserves a whole separated article but briefly… Houdini is a set of low-level APIs that exposes access to CSSOM (CSS Object Model) enabling developers to extend CSS by hooking into the styling and layout process of a browser’s engine. With this set of interfaces, only two restrictions come into play: your imagination and the browser support presented on “Is Houdini ready yet‽" by Surma. To find out about the power of Houdini I highly encourage you to check “CSS Houdini Experiments” by Vincent De Oliveira. “CSS Houdini & The Future of Styling by Una Kravets” is great and up-to-date introduction video. I would be careful using it in production yet, but it is a space to watch.

Feature query with @supports #

If you are wondering if the browser support for features listed above is sufficient, I have something that will give you a better motivation to start using them. Native feature detection that I elaborated on in my article “Native feature detection with CSS.supports() API” is a CSS / JS native way to check if your browser supports feature of interest. Look how cool it is!

@supports (display: grid) {
  .app {
    display: grid;
    grid-template-columns: repeat(2, 1fr);
    grid-gap: 1rem;
  } 
}

Preprocessors and the CSS-in-JS drama #

I mentioned in the foreword that, I regret neglecting my passion for CSS for the last few years. The CSS-in-JS drama is a thing that I am glad I missed. My take! If you need one, styled-components, Emotion and css-modules are safe bets to work with and I will pick one of these when I need it. For now, I don’t need it and I am happy.

CSS preprocessors like Sass, LESS and Stylus are not dead yet but slowly losing popularity simply because CSS by itself is getting much more powerful. I see a significant amount of people working with PostCSS that deserves to make it to this list. Probably one of the most exciting recent events from this space is the announcement of Sass modules. Miriam Suzanne posted recently “Introducing Sass Modules” on CSS Tricks that explains the concept (shouldn’t be alien for JavaScript developers familiar with modules).

Extra stuff #

I lot more stuff could be included on this list but I didn’t want to make this article miles long. I highly encourage you to follow Mozilla Developer where team members dig into obscure features of CSS. Another cool resource to watch to quickly catch up some freshness is a great talk by Una Kravets and Adam Argyle “Next-generation web styling” from Chrome Dev Summit 2019.

Keep on building beautiful things #

I hope that this list helped you out to modernize stylesheets of your current project. Let me know what is your favorite modern piece of CSS. Keep on building beautiful things and I will see you next time. Peace!
